    • What is a programming paradigm?

    Programming paradigms are ways in which we can organise programming languages depending on their roles.

    Different programming languages are better suited to different tasks. Some languages can multi-task and fit into multiple programming paradigms, others have single specific roles.

    Imperative Programming

    Imperative programming is a detailed, very specific set of instructions given to the computer to execute. It describes exactly how the program works, step by step.

    Procedural Programming

    Procedural programming is derived from imperative programming. This creates subdivisions within your functions to expand on those functions features.

    Procedural programming example:

    1. Pour flour in a bowl
    2. Pour a couple eggs in the same bowl
    3. Pour some milk in the same bowl
    4. Mix the ingredients
    5. Pour the mix in a mold
    6. Cook for 35 minutes
    7. Let chill
